SEポン出し（SEPonDashi）

Adobe Premiere Pro 用 効果音ポン出しパネル — 作成の流れと導入手順

CEP 拡張機能 Premiere Pro 2025 / 2026 Windows 対応 macOS 対応 WAV / MP3

概要

SEポン出しは Adobe Premiere Pro 上で動作する効果音ポン出しパネルです。 WAV / MP3 ファイルをあらかじめ登録しておき、ボタンひとつで再生ヘッドの位置にタイムライン配置できます。

項目内容
対応 OSWindows 10 / 11、macOS 12 以降
対応アプリAdobe Premiere Pro 2025 / 2026（v25.0 以上）
種別CEP 拡張機能（パネル）
対応フォーマットWAV / MP3
データ保存先（Windows）%APPDATA%\SEPonDashi\config.json
データ保存先（macOS）~/Library/Application Support/SEPonDashi/config.json

作成の流れ

STEP 1 — CEP 拡張機能の構造設計

CEP（Common Extensibility Platform）は Adobe CC アプリ上で動作する拡張機能の仕組みです。 パネルは Chromium ベースのウェブページとして動作し、Node.js と ExtendScript を通じてアプリ本体を操作します。 

// CEP拡張機能の構成イメージ
CEP拡張機能 = ウェブページ（HTML/JS/CSS）
             + Node.js（ファイル操作）
             + ExtendScript（Premiere Pro API）

STEP 2 — manifest.xml の作成

拡張機能の ID、対応バージョン、パネルサイズなどを定義します。

<ExtensionManifest
  ExtensionBundleId="com.example.sepondashi"
  Version="12.0">
  <HostList>
    <!-- Premiere Pro 2025=25.x, 2026=26.x -->
    <Host Name="PPRO" Version="[25.0,27.0]"/>
  </HostList>
  <CEFCommandLine>
    <Parameter>--enable-nodejs</Parameter>  <!-- Node.js有効化 -->
  </CEFCommandLine>
</ExtensionManifest>
ℹ️ Host Name="PPRO" で Premiere Pro を対象に指定。--enable-nodejs で Node.js を有効化し、ファイルシステムへのアクセスを可能にします。

STEP 3 — パネル UI の作成（index.html / style.css）

Premiere Pro のダークテーマに合わせた UI を HTML/CSS で構築します。

UI 要素機能
トラック選択ボタン（A2〜A5）配置先オーディオトラックの選択
カテゴリタブSE をグループ分けして管理（追加・削除・名前変更）
SE ボタングリッド / リストクリックで配置、右クリックで試聴
ドロップゾーンファイルをドラッグ＆ドロップして登録
検索バーSE 名でリアルタイム絞り込み
ログエリア操作結果・エラーメッセージを表示

STEP 4 — メインロジックの実装（main.js）

CEP パネル内の JavaScript で SE 管理・UI 制御を実装します。

機能内容
config 管理SE リスト・カテゴリを config.json に保存。拡張機能フォルダ外に保存するためアップデートで消えない
ファイルドロップドロップされた WAV/MP3 を assets/audio/ にコピーして登録
SE 配置evalScript() 経由で ExtendScript の placeSEOnTimeline() を呼び出す
試聴（右クリック）Web Audio API でローカルファイルを再生
スキーマ移行旧バージョンの config 形式を自動的に v5 形式へ変換

STEP 5 — Premiere Pro 連携（hostscript.jsx）

ExtendScript で Premiere Pro の API を操作してタイムラインにクリップを配置します。

function placeSEOnTimeline(filePath, audioTrackIndex) {
  // 1. プロジェクトパネルにファイルをインポート
  app.project.importFiles([filePath], true, ...);

  // 2. ターゲット以外のトラックをロック（誤配置防止）
  audioTracks[i].setLocked(i !== audioTrackIndex);

  // 3. 再生ヘッド位置にクリップを配置（3段フォールバック）
  seq.overwriteClip(projectItem, ctiTime, ...);  // 1st
  seq.insertClip(projectItem, ctiTime, ...);     // 2nd
  app.sourceMonitor.overwriteAtTime(ctiTime);    // 3rd
}

STEP 6 — インストーラーの作成

Windows

レジストリに PlayerDebugMode = 1 を設定し、署名なし拡張機能を Premiere Pro が読み込めるようにしてから、拡張機能ファイルをコピーします。

:: PlayerDebugMode を有効化
reg add "HKCU\Software\Adobe\CSXS.12" /v PlayerDebugMode /t REG_SZ /d 1 /f

:: 拡張機能フォルダへコピー（config・audioは保持）
robocopy "%SRC%\js" "%DEST%\js" /E /IS /IT
macOS

defaults write コマンドで PlayerDebugMode を設定し、拡張機能ファイルをコピーします。

# PlayerDebugMode を有効化
defaults write com.adobe.CSXS.12 PlayerDebugMode 1

# 拡張機能フォルダへコピー（config・audioは保持）
cp -R "$SRC/js" "$DEST/"
ℹ️ アップデート時はコードファイル（js / jsx / css / CSXS / lib / index.html）のみ上書きし、config.json と音声ファイルは一切触りません。

STEP 7 — パッケージング・配布

OS 別に別パッケージとして配布します。

パッケージ対象インストーラー
SEPonDashi_Windows.zipWindows ユーザーinstall.bat
SEPonDashi_Mac.zipmacOS ユーザーinstall.command
⚠️ SEPonDashi_Mac.zip は必ず macOS 上で作成してください。
Windows で ZIP を作成すると .command ファイルの実行権限（+x）が失われ、ダブルクリック実行ができなくなります。
Mac での作成コマンド: zip -r SEPonDashi_Mac.zip SEPonDashi_Mac/

導入手順 — Windows

初回インストール

  1. SEPonDashi_Windows.zip を展開する

  2. install.bat をダブルクリックする
    「Windows によって PC が保護されました」→「詳細情報」→「実行」

  3. INSTALLED と表示されたら成功。何かキーを押して閉じる

  4. Premiere Pro を再起動する

  5. 「ウィンドウ」→「エクステンション」→「SEポン出し」でパネルを開く

アップデート

  1. 新しい SEPonDashi_Windows.zip を展開する

  2. install.bat をダブルクリックする

  3. UPDATED - SE data has been preserved. と表示されたら成功

  4. Premiere Pro を再起動する

✅ 登録済みの SE・カテゴリはすべて保持されます。アンインストール不要です。

アンインストール

  1. uninstall.bat をダブルクリックする

  2. 1 Keep data（推奨）または 2 Full delete を選択する

  3. Premiere Pro を再起動する

導入手順 — macOS

初回インストール

  1. SEPonDashi_Mac.zip を展開する

  2. install.command を 右クリック → 「開く」 を選択する
    「開発元を確認できない」警告 → 警告ダイアログの「開く」をクリック（初回のみ）

  3. ターミナルが開き、INSTALLED と表示されたら成功。Enter で閉じる

  4. Premiere Pro を再起動する

  5. 「ウィンドウ」→「エクステンション」→「SEポン出し」でパネルを開く

ℹ️ 初回のセキュリティ確認が完了すると、2回目以降はダブルクリックで実行できます。

アップデート

  1. 新しい SEPonDashi_Mac.zip を展開する

  2. install.command を右クリック → 「開く」

  3. UPDATED - SE data has been preserved. と表示されたら成功

  4. Premiere Pro を再起動する

✅ 登録済みの SE・カテゴリはすべて保持されます。アンインストール不要です。

アンインストール

  1. uninstall.command を右クリック → 「開く」

  2. 1 Keep data（推奨）または 2 Full delete を選択する

  3. Premiere Pro を再起動する

よくある質問

パネルが「ウィンドウ → エクステンション」に表示されない
install.bat（または install.command）を再実行し、Premiere Pro を完全終了してから再起動してください。 Premiere Pro のバージョンが 25.0（2025）以上であることも確認してください。
効果音をクリックしてもタイムラインに配置されない
Premiere Pro でシーケンスが開いているか確認してください。 また、選択中のトラック（A2〜A5）がそのシーケンスに存在するか確認してください。 パネル下部のログエリアにエラーメッセージが表示されている場合は内容を確認してください。
アップデートしたら SE が消えた
別フォルダに展開した ZIP のインストーラーを実行した可能性があります。 uninstall（Keep data）でアンインストール後、再インストールし、バックアップの config.json を復元してください。
Mac で install.command をダブルクリックするとテキストが開く
右クリック → 「開く」を選択してください。 初回のセキュリティ確認が完了すると、以降はダブルクリックで実行できます。 「開く」が表示されない場合は「システム設定」→「プライバシーとセキュリティ」→「このまま開く」を選択してください。
Windows で作った ZIP を Mac ユーザーに配布しても大丈夫？
Windows で作成した ZIP は .command ファイルの実行権限が失われます。 Mac ユーザーがダブルクリックしてもテキストエディタが開くだけになります。 SEPonDashi_Mac.zip は必ず macOS 上で「zip -r SEPonDashi_Mac.zip SEPonDashi_Mac/」コマンドで作成してください。

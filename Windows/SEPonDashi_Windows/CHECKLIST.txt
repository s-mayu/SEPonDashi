  フォルダ構成の確認
  □ SEPonDashi_Windows/ に以下が揃っているか
      install.bat
      uninstall.bat
      README.txt
      CHECKLIST.txt
      SEPonDashi/
        CSXS/manifest.xml
        assets/audio/     ← 空であること（音声ファイルは含めない）
        config/config.json ← デフォルト（空の seList）であること
        css/style.css
        index.html
        js/main.js
        jsx/hostscript.jsx
        lib/CSInterface.js

  ファイル内容の確認
  □ config/config.json が空の seList であること（ユーザーデータが混入していないか）
  □ assets/audio/ に音声ファイルが入っていないこと
  □ js/main.js が最新バージョンであること
  □ .sh / .command ファイルが含まれていないこと（Windows専用パッケージ）

  動作確認（出荷前に実機テスト推奨）
  □ 新規マシンで install.bat を実行 → Premiere 再起動 → パネルが開くこと
  □ 既存インストール済みマシンで install.bat を実行 → config.json が保持されること
  □ 既存インストール済みマシンで install.bat を実行 → 音声ファイルが保持されること

  想定トラブルと対処
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  【ウィンドウ → エクステンション に表示されない】
  原因: PlayerDebugMode が設定されていない / 未反映
  対処: install.bat を再実行 → Premiere Pro を完全終了して再起動

  【install.bat を実行するとすぐ閉じる】
  原因: エラーが発生してスクリプトが終了している可能性
  対処: install.bat を右クリック →「管理者として実行」を試す

  【「INSTALLATION FAILED」と表示される】
  原因: フォルダ作成失敗 / ファイルコピー失敗
  対処:
    1. install.bat を右クリック →「管理者として実行」
    2. Premiere Pro が起動中であれば終了してから再実行
    3. ウイルス対策ソフトが bat ファイルをブロックしていないか確認

  【パネルは表示されるが効果音が配置されない】
  原因: シーケンスが開いていない / 対象トラックが存在しない
  対処:
    1. Premiere Pro でシーケンスを開く
    2. パネル上部のトラック選択（A2〜A5）がシーケンスに存在するか確認
    3. パネル下部のログ欄のエラーメッセージを確認

  【アップデート後に SE が消えた】
  原因: 別の場所に展開した ZIP の install.bat を実行した可能性
  対処:
    - アンインストール前にバックアップを作成していた場合:
      SEPonDashi_backup/config.json を
      %APPDATA%\SEPonDashi\config.json
      へコピーして Premiere Pro を再起動

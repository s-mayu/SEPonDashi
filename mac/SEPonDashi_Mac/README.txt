================================================================
  SEポン出し（SEPonDashi）
  Premiere Pro 用 効果音ポン出しパネル  【macOS 版】
================================================================

  動作環境
  ─────────────────────────────
  OS           : macOS 12 以降
  Premiere Pro : 2025 / 2026（バージョン 25.0 以上）

================================================================


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  初回インストール手順
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. 受け取った ZIP ファイルを展開します

  2.「install.command」を右クリック（または Control + クリック）し、
     「開く」を選択します

     ▼ 「開発元を確認できない」という警告が出た場合
       → 警告ダイアログ内の「開く」ボタンをクリックしてください
       ※ macOS のセキュリティ機能によるものです（初回のみ）

  3. ターミナルが開き、自動でインストールが進みます
     「INSTALLED」と表示されたら成功です
     Enter キーを押して閉じてください

  4. Premiere Pro を再起動します

  5. Premiere Pro のメニューから
     「ウィンドウ」→「エクステンション」→「SEポン出し」
     を選択するとパネルが開きます

  以上で完了です。


  ▼「開く」が表示されない場合の代替手順
  ─────────────────────────────
  1. アップルメニュー →「システム設定」→「プライバシーとセキュリティ」
  2. 画面下部に「"install.command"はブロックされました」と表示されている
     → 「このまま開く」をクリック
  3. 管理者パスワードを入力して許可します
  ─────────────────────────────

  アップデート手順
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. 新しいバージョンの ZIP を展開します

  2.「install.command」を右クリック →「開く」を選択します
     「UPDATED - SE data has been preserved.」と表示されれば成功です

  3. Premiere Pro を再起動します

  ※ 登録済みの効果音（SEリスト・カテゴリ）は保持されます
  ※ 古いバージョンのアンインストールは不要です


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  アンインストール手順
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1.「uninstall.command」を右クリック →「開く」を選択します

  2. 削除オプションを選択します

       1  Keep data（データを保持）  ← 推奨
          登録済み SE データと音声ファイルをデスクトップに
          バックアップしてから削除します

       2  Full delete（完全削除）
          すべてのデータを削除します（元に戻せません）

       3  Cancel（キャンセル）

  3. Premiere Pro を再起動します


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  基本的な使い方
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  効果音を登録する
    WAV / MP3 ファイルをパネル下部のドロップゾーンへドラッグ＆ドロップ

  効果音をタイムラインに配置する
    効果音ボタンをクリック
    → 再生ヘッドの位置に配置されます

  配置先トラックを選ぶ
    パネル上部の A2 ～ A5 ボタンで配置先を選択

  カテゴリを使う
    「＋」ボタンでカテゴリを追加
    効果音ボタンをカテゴリタブへドラッグして移動

  効果音を検索する
    検索バーに文字を入力してリアルタイム絞り込み

  効果音を試聴する
    効果音ボタンを右クリック（再度右クリックで停止）

  並び替える
    同じカテゴリ内で効果音ボタンをドラッグ＆ドロップ

  すべての SE を一覧表示する
    「すべて」タブで全カテゴリのSEを横断表示

  よくある質問（FAQ）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Q.「開発元を確認できない」という警告が毎回出ます
  A. 初回のみ右クリック →「開く」が必要です。
     一度許可すると、以降はダブルクリックで実行できます。

  Q. ダブルクリックしたらテキストが開いてしまいます
  A. 初回は右クリック →「開く」で実行してください。
     macOS のセキュリティ確認が完了すると、次回からダブルクリックで動作します。

  Q. Windows 版はありますか？
  A. Windows 版は別パッケージ（SEPonDashi_Windows.zip）をご利用ください。
     このパッケージは macOS 専用です。

  Q. インストール後に既存データはどうなりますか？
  A. config.json（SEリスト・カテゴリ設定）と
     assets/audio/ の音声ファイルは保持されます。

  Q. PlayerDebugMode とは何ですか？
  A. Adobe Premiere Pro が署名なしの拡張機能を
     読み込むために必要な設定です。
     install.command が自動で設定します。
     ※ この設定はお使いのユーザーアカウントのみに適用されます。

  Q. 「ウィンドウ → エクステンション」にパネルが表示されません
  A. 下記「トラブルシューティング」を参照してください。

  トラブルシューティング
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  【パネルが「ウィンドウ → エクステンション」に表示されない】

    手順 1: install.command を再実行して Premiere Pro を再起動する
    手順 2: Premiere Pro のバージョンを確認する
            「ヘルプ」→「Premiere Pro について」
            バージョン 25.0（2025）以上が必要です
    手順 3: 拡張機能フォルダにファイルが存在するか確認する
            Finder で以下のパスを開いて SEPonDashi フォルダがあるか確認
            ~/Library/Application Support/Adobe/CEP/extensions/
    手順 4: ターミナルで PlayerDebugMode を確認する
            defaults read com.adobe.CSXS.12 PlayerDebugMode
            ※「1」と表示されれば設定済みです
    手順 5: Premiere Pro を完全終了して再起動する

  【install.command を開くとテキストが表示されて実行されない】

    対処: 右クリック（または Control + クリック）→「開く」を選択する
          macOS のセキュリティ確認ダイアログで「開く」をクリック

  【効果音をクリックしてもタイムラインに配置されない】

    確認 1: Premiere Pro でシーケンスが開いているか
    確認 2: 配置先トラック（A2〜A5）がシーケンスに存在するか
    確認 3: パネル下部のログ欄のエラーメッセージを確認

  【登録した効果音がパネルに表示されなくなった】

    音声ファイルを移動・削除すると表示されなくなります。
    ファイルを元の場所に戻すか、再登録してください。

  【アップデート後に動作がおかしい】

    Premiere Pro を完全終了して再起動してください。
    解消しない場合は uninstall.command（Keep data）でアンインストール後、
    install.command で再インストールしてください。
